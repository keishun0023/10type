# kokorift ファネル再設計 仕様ドキュメント

作成日: 2026-06-12
目的: 広告訴求の変更（AIカウンセリング軸）に合わせて、診断〜ペイウォールのファネルを再設計する。
このドキュメントは、別セッションのClaudeに実装プランを立てさせるための設計指示書。

---

## 1. 背景とこれまでの経緯

### プロダクト概要
- **kokorift（ココリフト）**: 生きづらさを扱うセルフCBTプログラム
- 無料診断（現在36問）で「恐れ4軸 × 防衛3軸（接近回避/能動受動/抑制表出）」のスコアから内在化タイプ（8種）を判定
- 有料プログラム（3,980円・税込・買い切り）: 診断結果とオンボーディング回答をもとに、AIが30日間のデイリーミッション（認知ワーク＋行動ワーク）を個別生成
- 技術: Next.js App Router / Supabase / Stripe Checkout / Claude API
- 生成は2フェーズ: preview（Day1〜8、課金前）/ full（30日、課金後）

### 現在のファネル
```
広告 → LP → 無料診断(36問) → 簡易結果画面 → オンボ質問(別画面) → 深掘り質問 → building(生成待ち) → ペイウォール → Stripe決済 → ダッシュボード
```

### 計測実績
- 診断完了 → オンボ質問完了: 中間画面削除後 62%（削除前 27%）
- ペイウォール到達単価: 約78円
- 課金CV: まだ0（サンプル少）
- Metaピクセル2本: 875620858900566（全イベント）+ 537934028825464（ペイウォール到達ViewContent含む全イベント）

### 広告戦略の転換（今回の起点）
- 旧訴求:「生きづらさのタイプがわかる無料診断」→ 興味本位クリックが多い
- **新訴求:「その"生きづらさ"、AIカウンセリングで解消。CBT・認知行動科学に基づいたカウンセリングを自宅で体験」**
- ターゲット: 「カウンセリングを検討したことがあるが止まっている人」（25〜35歳中心、慢性的な生きづらさ、料金・予約・対人ハードルで踏み切れていない層）
- CTA: 「まずは無料で診断」→「まずは無料で体験」

### 核となるポジショニング（カウンセリング再フレーム）
- カウンセリングの対話の役割は「認知のクセに気づくこと」。変化を作るのはセッション間の日々の実践（CBTホームワーク）であることが研究で示されている
- このプログラムは「気づき（認知ワーク）」と「実践（行動ワーク）」の**両方を毎日**届ける。週1の対話より高頻度で認知にアプローチできる
- つまり「カウンセリングの劣化版」ではなく「カウンセリングが効く理由の部分を、毎日・自分のペースで・1/3以下の価格で」という提案

## 2. 新ファネル設計

```
広告 → LP → 無料診断(36問＋悩み系質問を統合) → 簡易結果画面(タイプ＋個別化された方向性＋CTA) → ペイウォール(思想→中身→比較→プレビュー→価格) → Stripe決済 → 初回セットアップ(利用条件質問) → ダッシュボード
```

### 変更点サマリ
1. オンボ質問のうち「悩みの構造」系を診断本体に統合
2. 簡易結果画面に「あなたへの方向性」セクションを追加（決定論テンプレートで個別化）
3. 別画面のオンボ質問・深掘り質問・building画面は廃止（または大幅縮小）
4. ペイウォールを「方向性に納得した人向けの実行手段の説明」として再構成
5. 「利用条件」系の質問は課金後の初回セットアップに移動

## 3. 詳細設計

### 3-1. 診断への質問統合

オンボ質問の仕分け:

**診断に統合する（「悩みの構造」の一部として自然なもの）:**
- しんどい場面（difficultScene）
- その詳細（difficultDetail / difficultFreeText）
- つらさレベル（distressLevel）
- 変わりたい/受け入れて楽になりたい（changeOrientation）

**課金後の初回セットアップに移動する（「利用条件」の質問）:**
- 1日に使える時間（dailyTime）
- 生活リズム（lifestyle）
- ベストタイミング（bestTiming）

設計意図: 診断の文脈で聞かれる質問は「自分を理解してもらうための入力」として自然に受け入れられる。利用条件の質問は診断の没入感を壊すので、課金後に「あなたに合わせて調整します」として聞く方が自然。

注意: dailyTimeは現在buildSchedule()でスケジュール構築に使われている。課金後に聞く形に変えるなら、preview生成時はデフォルト値を使い、初回セットアップ後にfull生成へ反映する設計にする。

### 3-2. 簡易結果画面の「方向性」セクション

- **生成方式: 決定論テンプレート（AI生成は使わない）**
  - 理由: 即時表示・コストゼロ・品質安定。AI生成の本領は課金後に取っておくことで「無料でこれなら有料はもっとすごい」という期待構造を作る
- 出し分けの軸: タイプ（8種）× しんどい場面 × 変化志向（変わりたい/楽になりたい）
- 内容: 「あなたの場合、まず◯◯から始めるのが効果的」という方向性の提示。タイプ説明で終わらず「で、どうすればいいか」の入口を見せる
- このセクションの役割: 「自分のことをわかってもらえた」体験の完成 ＋ プログラムへの橋渡し

### 3-3. CTA設計

- **CTAに価格は含めない**
  - 理由: 結果画面の役割は体験の完成。価値の説明（カウンセリング比較等）をする前に価格だけ見せると、文脈なしの3,980円になる。ペイウォールで価値説明→価格の順で納得を作る
- 文言イメージ: 「この方向で進める」「プログラムの内容を見る」程度のテンション

### 3-4. ペイウォールの再構成

訪問者は「方向性に納得してCTAを押した人」。役割は「興味を引く」ではなく「**この方向性を実行する手段の説明**」。

セクション構成（順番が重要）:

1. **方向性の再掲** — 結果画面からの連続性。「この方向で進めるなら」
2. **思想（設計の種明かし）** — 最初に置く。直前の「方向性への納得」を「まぐれじゃない」と証明する役割
   - 生きづらさは「恐れ」(4軸)と「防衛」(3軸)の組み合わせで構造が人によって違う
   - だから万人向けメソッドではなく、構造ごとにアプローチを変えるべき
   - さらに「変わりたいか、楽になりたいか」で認知と行動の配合を変える
   - あなたの30日は、この設計エンジンがあなたの診断結果から組んだもの
   - 効果: 占いっぽい安っぽさとの差別化、信頼の獲得
3. **具体的な中身** — 毎日1つ届くワーク、認知/行動の2種類、1日5〜10分、ダッシュボード
4. **カウンセリング比較（再フレーム）**
   - 「カウンセリング=話すこと」だと思っていませんか?
   - 週1回の対話で気づく → 毎日の認知ワークで気づく
   - 次のセッションまで放置 → その日のうちに小さく試す
   - 月1〜2万円の継続費用 → 3,980円・買い切り
   - 予約して通って話す → スマホで自分のペースで
5. **3日間プレビュー**（既存を流用、個別化されたもの）
6. **価格 + 決済CTA + FAQ**

表現上の注意:
- 「うつが治る」等の疾患への効果断定はNG（薬機法・Meta審査）。「生きづらさ」は抽象概念なので「解消」は可
- 対話型チャットと誤解させない（実態は30日プログラム）

### 3-5. 廃止・移動するもの

- オンボ質問画面（悩み系は診断へ、条件系は課金後へ）
- 深掘り質問画面（diffucultDetail等を診断に統合する場合）
- building画面（生成はバックグラウンド化を検討。プレビュー生成のタイミング設計は実装時に要検討: 診断完了時に裏で開始する等）

## 4. 実装上の注意

- 既存ファイル:
  - 診断: `data/questions.ts`、診断フロー本体
  - 簡易結果: `components/ResultScreen.tsx`（現在: タイプ表示→あるある→なぜそうなる→CTA→フッター）
  - オンボ/ペイウォール: `app/program/page.tsx`（screen state: onboarding → deepdive → building → pricing）
  - 生成: `lib/generatePlan.ts`（preview/full、onboardingを使用）、`lib/schedule.ts`（dailyTime使用）、`lib/prompt.ts`
  - 計測: `lib/pixel.ts`（両ピクセル発火。イベント: PageView / StartTrial / CompleteRegistration(診断完了) / ViewContent(ペイウォール到達・537のみ) / InitiateCheckout(プラン選択)）
- ペイウォールのv2デザインが `app/program/v2/page.tsx` に保存されている（宣伝くさいと判断して本番からは外した経緯あり。トーンの参考として注意）
- 段階的リリースを推奨: (1)結果画面の方向性セクション＋CTA変更 → (2)ペイウォール再構成 → (3)質問統合とオンボ廃止。一度に全部変えると効果測定ができない

## 4-2. 確認済みの現状事実（Q&Aより）

- **生成アーキテクチャ**: 配合エンジン（lib/blending.ts の buildProgramConfig、決定論）が部品配合を決め、buildSchedule が30日の地図（日×部品×kind）を決定論で組む。AIは枠の中の文面のみ書く。preview は Day1〜8 生成（ペイウォールで見せるのは3日分、残りは課金直後にすぐ本物が見えるためのバッファ）。full は部品（恐れ×kind）ごとのバッチで並列生成
- **防衛は3軸**: D_APP（接近回避）/ D_ACT（能動受動）/ D_EXP（抑制表出）
- **タイプは8種**: good_child / people_pleaser / over_considerate / perfectionist / over_carrier / abandonment / distancer / worrier
- **価格**: APIには light(980)/standard(3,980)/premium(8,980) が残るが、UIで推しているのは standard 3,980 のみ（premium ボタン残存、light 非表示）
- **78円/ペイウォール到達は旧訴求（診断系クリエイティブ）の数字**。AIカウンセリング軸の新クリエイティブは未配信。「到達単価は良いのにCV 0」→層の質が原因と判断→本再設計、という経緯
